# 003 MVP 范围

## 日期

2026-03-25

## 状态

已采纳

## 决策

`Note2Video` 的第一版 MVP 聚焦于一个范围收敛但流程完整的讲解视频流水线。

## 范围内

- 输入：`pptx`
- 导出页面图片
- 提取每页备注
- 将备注整理为口播脚本
- 使用一个主 TTS Provider 生成配音
- 生成 `srt` 字幕
- 渲染最终 `mp4`
- 支持 `16:9` 与 `9:16`
- 同时提供一键命令与逐步子命令

## 核心命令

- `note2video build`
- `note2video extract`
- `note2video voice`
- `note2video subtitle`
- `note2video render`

## MVP 范围外

- PowerPoint 动画完整保真
- 高级多角色配音
- 复杂音乐混音
- 自动补画面或视觉增强
- 剪映集成
- PowerPoint 插件形态
- Web SaaS 形态
- 多语言本地化工作流

## 原因

这个范围足够小，可以快速开发和测试；同时又足够完整，能验证产品价值：

- 用户可以从一个 `pptx` 文件走到最终视频
- 每一步中间产物都可查看、可调试
- CLI 既能服务个人使用，也能服务后续自动化封装

## MVP 质量标准

如果满足以下条件，就可以认为 MVP 成功：

- 一条命令能够从带备注的 PPT 生成可用视频
- 中间输出清晰、可读、可调试
- 重复执行结果稳定
- 项目结构已经适合后续 skill 封装

## 当前实现补充

在 `extract` 阶段，除了 JSON 产物外，还应输出逐页与汇总 `.txt` 文件，方便人工复制、审阅和继续编辑。

## 后续开发优先级

1. `extract`
2. 备注与清单结构
3. 第一个 TTS Provider
4. 字幕生成
5. 视频渲染
6. `build` 顶层编排
